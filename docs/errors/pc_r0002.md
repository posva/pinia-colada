# PC_R0002

**Level:** error (development only) · **Area:** runtime

## What happened

The query cache `ref` was **reassigned** (e.g. `queryCache.caches.value = somethingElse`) instead of being mutated in place. The cache must keep the same underlying object so Pinia Colada and its plugins can track entries. Reassigning it would break in production.

## How to fix it

Mutate the existing cache rather than replacing it. Use the cache actions (`get`, `set`, `setQueryData`, `invalidateQueries`, etc.) instead of swapping out the internal map.

```ts
const queryCache = useQueryCache()

// ❌ replaces the reactive value
// queryCache.caches.value = new Map()

// ✅ mutate through the public API
queryCache.setQueryData(['todos'], [])
```

## Common causes

- Resetting the cache with an assignment instead of the provided actions.
- A devtools or test helper assigning to the internal ref.
