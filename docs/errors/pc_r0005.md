# PC_R0005

**Level:** warning (development only) · **Area:** runtime

## What happened

A composable created with `defineMutation()` was called **outside of a component or effect scope**. Without a scope, the mutation's reactive effects are never cleaned up, which can lead to memory leaks.

## How to fix it

Call the defined mutation from a place that has an effect scope:

- inside a component `setup()`
- inside an `effectScope()`
- inside a Pinia store setup

```ts
import { defineMutation, useMutation } from '@pinia/colada'

const useCreateTodo = defineMutation(() => useMutation({ mutation: createTodo }))

// ✅ called inside setup()
function MyComponent() {
  const { mutate } = useCreateTodo()
}
```

## Common causes

- Calling the defined mutation at module top-level.
- Calling it inside an async callback after `await`, which loses the active scope.
