# PC_R0006

**Level:** warning (development only) · **Area:** runtime

## What happened

`useMutationCache()` was called outside of an injection context (a component `setup`, a store, or a navigation guard). Vue's `inject` only works during these contexts, so it will additionally emit its own warning about `inject` being used incorrectly.

## How to fix it

Call `useMutationCache()` only where injection is allowed:

- inside a component `setup()`
- inside a Pinia store setup
- inside a Vue Router navigation guard
- inside a composable that is itself called from one of the above

```ts
import { useMutationCache } from '@pinia/colada'

function useThing() {
  // ✅ called from a composable used in setup()
  const mutationCache = useMutationCache()
  // ...
}
```

If you genuinely need the cache outside of a component, pass the `pinia` instance explicitly: `useMutationCache(pinia)`.

## Common causes

- Calling `useMutationCache()` at module top-level.
- Calling it inside an async callback after `await`, which loses the injection context.
