# PC_R0009

**Level:** error (development only) · **Area:** runtime

## What happened

A mutation entry was **reused**: `mutationCache.mutate()` was called on an entry that already has an ongoing or completed request. Each mutation call needs a freshly ensured entry.

## How to fix it

If you are manually driving the mutation cache, ensure the entry before each mutation:

```ts
const mutationCache = useMutationCache()

// ✅ ensure first, then mutate
mutationCache.mutate(mutationCache.ensure(entry, vars))
```

If you are not calling `mutationCache.mutate()` directly, this is likely an internal bug — please open an issue with a boiled-down reproduction:

- <https://github.com/posva/pinia-colada/issues>

## Common causes

- Calling `mutate()` twice on the same entry without re-ensuring it.
- Holding onto a previously used entry instead of creating a new one.
