# PC_R0003

**Level:** error (thrown, development only) · **Area:** runtime

## What happened

`useQuery()` (or a related helper) was called with an **empty array** as its `key`. A query key must uniquely identify the query and therefore needs at least one element.

## How to fix it

Provide a key with at least one element:

```ts
import { useQuery } from '@pinia/colada'

// ❌ empty key
// useQuery({ key: [], query: getTodos })

// ✅ at least one element
useQuery({ key: ['todos'], query: getTodos })
```

If the key depends on a value that may be empty, guard the query with `enabled` or compute a non-empty key.

## Common causes

- A computed key that returns `[]` before its dependencies are ready.
- Spreading an empty array into the key.

