# PC_R0008

**Level:** error (development only) · **Area:** runtime

## What happened

A mutation entry was passed to `mutationCache.mutate()` **before being ensured**. An entry must be ensured first so it has its variables and options set up.

## How to fix it

If you are manually driving the mutation cache, always ensure the entry before mutating it:

```ts
const mutationCache = useMutationCache()

// ✅ ensure first, then mutate
mutationCache.mutate(mutationCache.ensure(entry, vars))
```

If you are not calling `mutationCache.mutate()` directly, this is likely an internal bug — please open an issue with a boiled-down reproduction:

- <https://github.com/posva/pinia-colada/issues>

## Common causes

- Calling `mutationCache.mutate(entry)` without `ensure()`.
- Reusing an entry object that was never ensured.
