# PC_R0004

**Level:** error (thrown, development only) · **Area:** runtime

## What happened

`entry.refresh()` or `entry.fetch()` was called on a query entry that has no options attached. An entry must be initialized (via `useQuery()` or `queryCache.ensure()`) before it can be refreshed or fetched.

## How to fix it

This is almost always an internal bug rather than something in your application code. If you are manually working with the query cache, make sure the entry was ensured before calling `refresh()`/`fetch()`.

If you can reproduce it, please open an issue with a boiled-down example:

- <https://github.com/posva/pinia-colada/issues>

## Common causes

- Calling `refresh()`/`fetch()` on an entry retrieved before it was ensured.
- An internal edge case — reports are welcome.
