# PC_R0011

**Level:** warning (development only) · **Area:** runtime

## What happened

`useInfiniteQuery()` tried to load a page (next or previous) but the query entry was **not found in the cache**. This usually means the query has not been mounted yet, or its key changed.

## How to fix it

Make sure the infinite query is mounted with `useInfiniteQuery()` before calling `loadNextPage()` / `loadPreviousPage()`, and that the key has not changed out from under it:

```ts
const { loadNextPage } = useInfiniteQuery({
  key: ['projects'],
  query: ({ pageParam }) => fetchProjects(pageParam),
  initialPageParam: 0,
  getNextPageParam: (lastPage) => lastPage.nextCursor,
})

// ✅ call after the query is mounted
loadNextPage()
```

## Common causes

- Calling `loadNextPage()`/`loadPreviousPage()` before the component using the query has rendered.
- The query `key` changed, evicting the old entry from the cache.
