# PC_R0010

**Level:** error (thrown, development only) · **Area:** runtime

## What happened

`useInfiniteQuery()` tried to load the **previous** page, but `getPreviousPageParam` is not defined in its options. Without it, Pinia Colada cannot compute the parameter for the previous page.

## How to fix it

Define `getPreviousPageParam` in the options passed to `useInfiniteQuery()` whenever you call `loadPreviousPage()`:

```ts
import { useInfiniteQuery } from '@pinia/colada'

useInfiniteQuery({
  key: ['projects'],
  query: ({ pageParam }) => fetchProjects(pageParam),
  initialPageParam: 0,
  getNextPageParam: (lastPage) => lastPage.nextCursor,
  // ✅ required to load previous pages
  getPreviousPageParam: (firstPage) => firstPage.prevCursor,
})
```

If you only ever paginate forward, do not call `loadPreviousPage()`.

## Common causes

- Calling `loadPreviousPage()` on a query that only defines `getNextPageParam`.
